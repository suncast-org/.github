Thank you for your interest in contributing! SUNCAST welcomes contributions from researchers, students, and developers worldwide. This document provides the practical steps required to contribute code, documentation, or new tools.

---

## üß∞ Contribution Workflow (For All Repositories)

1. **Fork the repository** to your personal GitHub account.  
2. **Create a feature branch** for your work:  
   ```bash
   git checkout -b my-feature
   ```
3. **Make your changes** (code, docs, tests, examples).  
4. **Commit with clear messages**:  
   ```bash
   git commit -m "Add new SK threshold example"
   ```
5. **Push your branch** to your fork:  
   ```bash
   git push origin my-feature
   ```
6. **Open a Pull Request** to the repository under `suncast-org`.  
7. Participate in the review process.

---

## üå± Starting New Projects

If you want to propose a new tool or workflow:

### **Option A: Independent development (recommended for new ideas)**
1. Create the repository under your personal GitHub space.
2. Develop and test your idea freely.
3. When the project is ready for broader use, request a **repository transfer** into the SUNCAST organization.

### **Option B: Start directly under SUNCAST (only for coordinated projects)**
If the project is part of a SUNCAST deliverable or collaborative effort, ask a maintainer to create a repository under `suncast-org` and follow the fork ‚Üí PR model.

## üß™ Standards & Expectations

- Code should follow the conventions of the language (Python or IDL).
- Documentation and examples are strongly encouraged.
- All code must use an OSI-approved open-source license.
- Tests are encouraged when appropriate.
- PRs should be small and focused.
